EGL Headless Rendering
----------------------
STATUS: WORKS (with workaround)

Rendering works with MUJOCO_EGL_DEVICE_ID=1 (Mesa software EGL).
NVIDIA hardware EGL (device 0) fails due to driver version mismatch:
  - Kernel module: 550.107.02
  - Userspace libs: 550.163.01 (upgraded when installing libnvidia-gl-550)
  - nvidia-smi reports: "Driver/library version mismatch"

Required environment variables for rendering:
  export MUJOCO_GL=egl
  export MUJOCO_EGL_DEVICE_ID=1
  export PYOPENGL_PLATFORM=egl
  export DISPLAY=''

Known Issues
------------
1. NVIDIA EGL device initialization fails — must use MUJOCO_EGL_DEVICE_ID=1 (Mesa software)
   - Root cause: libnvidia-gl-550 install upgraded userspace libs to 550.163.01 but kernel module is 550.107.02
   - Fix options: (a) reboot to load new kernel module, or (b) find matching 550.107.02 libnvidia-gl package
   - Software rendering is slower but functionally correct
   - CUDA/torch still works fine (only NVML/nvidia-smi affected)

2. robosuite downgraded from 1.5.2 to 1.4.1
   - LIBERO uses robosuite 1.4.x API (SingleArmEnv, etc.) which was removed in 1.5.x
   - VLA-0 eval code only uses load_controller_config, which exists in both versions

3. The "datasets" path warning is expected — demo datasets not needed for evaluation
